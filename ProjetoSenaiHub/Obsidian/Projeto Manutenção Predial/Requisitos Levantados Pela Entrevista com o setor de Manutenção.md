Sistema de Abertura de Chamados Para todo o Prédio da unidade do Senai.
A ideia é fazer um sistema q pode ser implantado em qualquer unidade existente e q existirá no Brasil, ou seja, deve ser adaptável as necessidades gerais das escolas da unidade Senai.

Ideia Geral de como vai ser estruturado:
Um sistema interno amigável em UI/UX, funcionará como uma rede social interna ao mesmo tempo q vai conter coisas de suma importância para o dia a dia da empresa (TOTVS Fluig será usado como inspiração de ecossistema)(apenas protótipo, o foco atual do projeto é a penas no sistema de chamados. a ideia será usada como argumento de escalabilidade do sistema)
Categorização dos usuários por grupos/papeis para melhor gestão das atribuições internas do sistema

Como é feito Atualmente:
Planilha do Excel, manual, verbal e papel.

Como deve ser:
Sem papel, sem fala, totalmente via sistema. O sistema no geral deve ser capaz de receber importações de arquivos csv e Excel(controles já existentes) para tornar mais fácil a importação de maiores bases de dados e exportação para relatórios.
Deve conter filtros para filtrar qualquer coisa q seja consultável no sistema, controles por data, funcionário, setor.

Como deve ser o sistema?
Cobrir todas as áreas do Senai.

Quem pode solicitar?
Funcionário da área, Ex: Se teve um problema na fiação do Bloco C, apenas o funcionário responsável pela manutenção do Bloco C porem solicitar.

Como deve ser a Solicitação?
- Código para controle interno de chamados e histórico
- Descritiva, descrever bem o problema
- Anexo de Prova (imagem, qualquer arquivo q comprove a falha).
- Grau de impacto, campo para descrição, tipo impossibilita o acesso a área, uso dos alunos/funcionários, inviabiliza o trabalho ou da pra ser virar do jeito q tá? (dps vai ser definido de maneira séria)
- Date e hora q está sendo feita a solicitação
- Por quem e para quem
- Prazo
- Oque foi danificado/faltando (vai ser definido de maneira mais detalhada)

Próxima Etapa: Área da manutenção recebe a solicitação e avalia mais a fundo o problema.

Oque ela vai fazer?
- Define o grau real da situação com base no material recebido da solicitação, define se é urgente, se e médio, se pode esperar.
- Delega funcionário q vai ser responsável pela tarefa
- Avalia se vai ser resolvido por serviço externo, equipe interna, especialista interno
- Se existe oq foi solicitado no estoque caso seja algo para substituição
- Define o nível de complexidade da tarefa, ex: instalar novas maquinas exige mais da rede elétrica, ou seja para isso acontecer exige uma melhoria conjunta desses setores.
- Se For algo feito por serviços externos qual empresa vai fazer, preço do serviço + Equipamentos(caso n seja manutenção)
- Se for precisar comprar algo encaminhar orçamento para a área financeira orçamento para aprovação
- Caso seja um projeto maior encaminhar para diretor ou alguém q tenha q aprovar o projeto

Quando o funcionário/Prestador de serviço efetuar o atendimento do chamado como ele deve ser encerrado?
- Comprovação q foi arrumado(imagem)
- Comprovantes de compra caso necessário
- Descrição do q foi feito, como foi feito para consulta caso o problema retorne(base do conhecimento)
- Instabilidades q foram presenciadas durante o serviço(ex, energia n aguenta nova maquina e será necessário solicitar ajuda do setor da elétrica para concluir a tarefa)
- Esse processo deve ter um responsável q acompanhará todo o trabalho q será efetuado referente ao chamado

Após a conclusão do chamado:
- Deve ser armazenado todo o histórico de chamados abertos com os anexos para consulta caso necessário para comprovações em auditorias futuras

Outras especificações técnicas como tecnologias e coisas do desenvolvimento serão abordadas futuramente.
